Concepts de base¶

Un "graphique" est strucutr de la manire suivante :

  • Figure est l'objet de plus haut niveau : "coquille externe" du graphique
  • Axes contient l'objet graphique en lui-mme
  • Axis est un axe du graphique (contrle des labels, xmin, xmax, etc.)
  • Artist est chaque lment dessin sur un graphique (ligne, titre, etc)
  • Canvas est la zone de trac du graphe

Les donnes tracer doivent tre de type array de numpy. (np.array)

Toute utilisation de matplotlib commence donc toujours par importer les modules matplotlib.pyplot et numpy sous la forme usuelle ci-dessous :

In [1]:
import matplotlib.pyplot as plt
import numpy as np

Le module pyplot cre automatiquement (et de faon transparente) les objets figure et axes ncessaires la construction d'un graphique. Chaque "lment" qu'on rajoute par la suite (ex : plt.plot() ou plt.title()) s'ajoute la figure et l'axe actif.

Pour visualiser la figure cre, il faut utiliser la commande plt.show()

Des conseils de bon usage¶

For non-interactive plotting it is suggested to use pyplot to create the figures and then the OO interface for plotting. https://matplotlib.org/api/api_overview.html

L'utilisation de pyplot est plus simple (mais permet de faire des choses moins prcises) que l'utilisation de matplotlib de faon oriente objet.

In [2]:
x = np.linspace(0, 2, 100)

plt.plot(x, x, label='linaire')
plt.plot(x, x**2, label='carr')
plt.plot(x, x**3, label='cube')

plt.xlabel('abscisses')
plt.ylabel('ordonnes')

plt.title("Graphique basique")

plt.legend()

plt.show()

Backend¶

matplotlib peut tre utilis dans beaucoup de situations diffrentes et ncessite des sorties diffrentes. (shell python avec des fentre pop up, Jupyter notebooks, graphiques embarqus dans des GUI comme wxpython ou pygtk , sorties d'images postscript).

Tous ces moyens sont des backends.

Le frontend est le code pour tracer le graphe. Le backend fait tout le travail "behind-the-scenes" pour produire la figure.

Il existe 2 types de backends:

  • user interface backends (pygtk, wxpython, tkinter, qt4, macosx) : ce sont les interactive backends.
  • hardcopy backends pour crer des fichiers images(PNG, SVG, PDF, PS) : ce sont les non-interactive backends.

Pour savoir comment configurer son Backend (si ncessaire), consulter prcisment la doc suivante : https://matplotlib.org/tutorials/introductory/usage.html#backends

Certains backends peuvent tre activs dans un Jupyter notebook avec les fonctions magiques %matplolib 'nom du backend' (il y a un backend spcial Jupyter mais cela ncessite l'installation d'un module iympl https://github.com/matplotlib/jupyter-matplotlib).

Dans un environnement de notebook, il peut tre intressant de lister les backends existants pour trouver celui qui convient notre besoin du moment (qui peut varier videmment). Il faut redmarrer le kernel pour changer de backend. Voir la liste ci-dessous :

In [3]:
%matplotlib -l

Available matplotlib backends: ['tk', 'gtk', 'gtk3', 'wx', 'qt4', 'qt5', 'qt', 'osx', 'nbagg', 'notebook', 'agg', 'svg', 'pdf', 'ps', 'inline', 'ipympl', 'widget']

A priori, mes choix vont dans cet ordre :

1- notebook

2- inline

3- qt

Avec le backend notebook, il est conseill de numroter les figures pour viter des soucis.

In [4]:
%matplotlib notebook

x = np.linspace(0, 2, 100)

plt.figure(42)

plt.plot(x, x, label='linaire')
plt.plot(x, x**2, label='carr')
plt.plot(x, x**3, label='cube')

plt.xlabel('abscisses')
plt.ylabel('ordonnes')

plt.title("Graphique basique")

plt.legend()

plt.show()

Mode interactif¶

Il est possible (avec un backend interactif) de commuter le mode interactif ou non :

  • plt.ion() : active le mode interactif : on peut modifier la figure " chaud". La commande plt.draw() peut tre ncessaire pour forcer le rafrachissment de la figure.
  • plt.ioff() : dsactive le mode interactif : la figure n'apparat que lorsque la commande plt.show() est excute. Cette commande bloque le script python en cours jusqu' ce que la fentre graphique soit close.

Pour connatre l'tat courant : plt.isinteractive()
